В шаблоне head.home вы можете разместить специфичные только для главной страницы  скрипты и стили. В этом варианте темы Шаблон является javascript загрузки изображения. Надо заметить, что шаблонизатору нет никакого дела, что в шаблоне - html или javascript, Выбор всегда находится в руках разработчика темы. 

Рассмотрим детально, что делает скрипт в шаблоне, для этого процитируем код с комментариями:

    $("#site-description").homeimage({
    // "#site-description - id контейнера, смотрите шаблон index, он находится сразу после панели навигации

      image: "$home.image",
      // image - адрес большой картинки , использован тег $home.image

      small: "$home.smallimage",
      // small - адрес картинки для мобильных (тег $home.smallimage), не обязательный аргумент, если отсутствует или пустая строка, то будет использована одна большая картинка для всех разрешений

      breakpoint: 768,
      // breakpoint - не обязательный параметр, по умолчанию = 768, это размер окна, при котором происходит смена картинки - для десктопов одна, для мобильных устройств - другая меньшая картинка. Для темы поумолчанию на фреймворке bootstrap это 768 и это значение не надо менять. Если вы делаете свою тему, то возможно, что ширина экрана будет отличаться. для таких редких случаев как раз и оставлена возможность указать размер для скрипта.

      addclass: "home-image"
      //addclass - не обязательный параметр, по умолчанию = home-image. Это имя css класса, который будет добавлен к #site-description Определение css класса находится в файле /js/litepubl/effects/css/homeimage.min.css
    });

Метод homeimage находится в файле /js/litepubl/effects/homeimage.min.js - поумолчанию он включен сборщиком js, поэтому, если вы хотите отключить загрузку картинок на главной, то вам следует удалить из сборщика этот файл и удалить из шаблона head.home скрипт инициализации . Для работы скрипта в сборщике css подключен файл /js/litepubl/effects/css/homeimage.min.css - в случае отключения про него тоже не забудьте (равно как и при подключении этой фичи к своей теме). 

Шаблон и скрипт коректно отработают (то есть на главной будет масштабирующаяся картинка), если в настройках главной (админка настройки/главная)   указать адрес (url) картинок.  Особенности скрипта:
- выбор размера картинки в зависимости от разрешения экрана, для мобильных устройств это значительно ускорит открытие сайта
- один шаблон для всех страниц, не нужно отдельно верстать страничку для главной, картинка будет только на главной

В версии litepublisher 5.73 были добавлены новые шаблоны $template.index.home, $template.index.post и $template.index.tag - шаблоны главной страницы, записи и рубрики/метки. Как правило к этим страницам предъявляются особые требования. В предыдущих версиях движка приходилось создавать отдельные темы для главной, записи и рубрик, что вызывало определенные неудобства во время разработки новой темы.
